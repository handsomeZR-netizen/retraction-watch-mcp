# Retraction Watch MCP

A local Model Context Protocol (MCP) server for conservative screening against the public Retraction Watch dataset distributed by Crossref.

This project is designed for local use: the public Retraction Watch CSV is downloaded into a local SQLite index, and person-screening queries stay on the user's machine.

## Status

- Version: `0.1.0`
- Runtime: Node.js `>=20`
- Transport: MCP `stdio`
- Supported platforms: Windows, macOS, Linux
- Data storage: local SQLite generated by `rw-import`

## What It Does

- Downloads the Crossref/GitLab Retraction Watch CSV into a local SQLite database.
- Exposes MCP tools for person screening, batch screening, DOI lookup, record lookup, match explanation, and source version reporting.
- Uses conservative, explainable matching. Email is reduced to a domain-only weak signal because the public Retraction Watch data has no email field.
- Returns all Retraction Watch notice types by default and labels `RetractionNature` explicitly.
- Separates likely/possible matches from `nearMisses`, and always warns that author-to-affiliation mapping is unavailable in the source data.

## What It Is Not

This is a screening aid, not an identity adjudication system.

A non-DOI match means the input resembles author and affiliation strings in Retraction Watch data. It does not prove the named person is the same individual, and it does not prove personal misconduct. Only DOI/PMID exact matches are returned as `confirmed`.

## Install From GitHub

```bash
git clone https://github.com/handsomeZR-netizen/retraction-watch-mcp.git
cd retraction-watch-mcp
npm ci
npm run build
```

Build artifacts are generated into `dist/`. The generated Retraction Watch SQLite database is not committed to the repository.

## Initialize Or Update Data

Run the importer before starting the MCP server:

```bash
npm run import
```

By default, the SQLite database is written to:

- Windows: `C:\Users\<you>\.retraction-watch-mcp\retraction-watch.sqlite`
- macOS/Linux: `~/.retraction-watch-mcp/retraction-watch.sqlite`

Use a custom path when needed:

```bash
npm run import -- --db-path ./data/retraction-watch.sqlite
```

The source CSV is currently about 64 MB, and the generated SQLite database is several hundred MB. Re-run the importer whenever you want to refresh the local data.

## Run The MCP Server

```bash
npm run mcp
```

With a custom database path:

```bash
npm run mcp -- --db-path ./data/retraction-watch.sqlite
```

You can also configure the database path with environment variables:

- `RW_MCP_DB_PATH`: exact SQLite database path
- `RW_MCP_DATA_DIR`: directory for the default database file

## MCP Client Configuration

Use an absolute path to the compiled server.

### Claude Desktop

```json
{
  "mcpServers": {
    "retraction-watch": {
      "command": "node",
      "args": ["D:\\\\path\\\\to\\\\retraction-watch-mcp\\\\dist\\\\index.js"],
      "env": {
        "RW_MCP_DB_PATH": "D:\\\\path\\\\to\\\\retraction-watch-mcp\\\\data\\\\retraction-watch.sqlite"
      }
    }
  }
}
```

### Cursor

`.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "retraction-watch": {
      "type": "stdio",
      "command": "node",
      "args": ["/absolute/path/to/retraction-watch-mcp/dist/index.js"],
      "env": {
        "RW_MCP_DB_PATH": "/absolute/path/to/retraction-watch.sqlite"
      }
    }
  }
}
```

### Claude Code

```bash
claude mcp add retraction-watch -- node /absolute/path/to/retraction-watch-mcp/dist/index.js
```

If you use a non-default database path, set `RW_MCP_DB_PATH` in your shell before launching the client, or pass `--db-path` in the configured command arguments.

## CLI Usage

Query directly without an MCP client:

```bash
npm run query -- --name "Ahsen Maqsoom" --institution "COMSATS University Islamabad"
```

With explicit data path:

```bash
npm run query -- --db-path ./data/retraction-watch.sqlite --name "Ahsen Maqsoom" --institution "COMSATS University Islamabad"
```

Optional global-style commands are available after linking the package locally:

```bash
npm link
rw-import
rw-query --name "Ahsen Maqsoom" --institution "COMSATS University Islamabad"
rw-mcp
```

## MCP Tools

- `screen_person`: screen one person using name, optional email, institution, DOI, PMID, and notice-type filter.
- `screen_batch`: screen multiple people.
- `lookup_record`: return one Retraction Watch record by `Record ID`.
- `lookup_doi`: return records whose original paper DOI or retraction notice DOI exactly matches the provided DOI.
- `explain_match`: explain how one query scores against one record.
- `get_source_versions`: return the local data snapshot and policy version.

## MCP Resources

- `rw://source-version`
- `rw://match-policy/current`
- `rw://record/{record_id}`

## Result Interpretation

Top-level result fields include:

- `verdict`: `confirmed`, `likely_match`, `possible_match`, or `no_match`
- `identityConfirmed`: `true` only for DOI/PMID exact matches
- `reviewRequired`: whether human review is required before consequential use
- `candidates`: likely/possible/confirmed matches
- `nearMisses`: rejected records with useful conflicting or weak evidence
- `warnings`: source and identity-matching limitations
- `manualReviewReasonCodes`: machine-readable review reasons
- `inputDiagnostics`: email-domain handling and source-data capability notes

## Data Source And Licensing

Primary data source:

`https://gitlab.com/crossref/retraction-watch-data/-/raw/main/retraction_watch.csv`

Crossref documents the GitLab repository as the best way to keep the Retraction Watch dataset current with `git clone` / `git pull`. This project currently downloads the raw CSV for a simple local workflow. The legacy Crossref Labs endpoint is not treated as a stable contract, and the Retraction Watch web search page is not used for programmatic access.

Crossref lists the Retraction Watch database as CC0. For published or shared research outputs, cite Retraction Watch/Crossref according to the Retraction Watch user guide.

## Troubleshooting

- `Local Retraction Watch database not found`: run `npm run import`, or set `RW_MCP_DB_PATH` to an existing database.
- MCP client starts but tools fail: make sure the client command points to `dist/index.js` and that `npm run build` has been run.
- Import is slow: the CSV and generated SQLite database are large; this is expected.
- Results look ambiguous: inspect `warnings`, `manualReviewReasonCodes`, `nearMisses`, and use `explain_match` for a single record.
